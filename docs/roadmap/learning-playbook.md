# Learning Playbook

Use this playbook to make each topic stick instead of just feeling familiar.

## Core Method

For each topic, move through this order:

1. intuition
2. formal definition or math
3. worked example
4. implementation
5. explanation from memory
6. quiz or error check

## Best Way To Learn A Daily Topic

- start by asking what problem this concept solves
- learn one clean mental model before going deep
- translate the concept into formulas, tensors, or steps
- write a tiny implementation or worked example the same day
- explain the concept back in your own words before ending

## Note-Taking Rules

- write notes as answers to questions, not as copied summaries
- capture one misconception and one corrected understanding
- write down what inputs, outputs, and failure modes look like
- if there is math, write the symbols and explain them in plain English

## Video Learning Rules

- watch with a purpose, not passively
- pause to predict what comes next
- stop after important sections and restate the idea
- convert the video into 3 to 5 bullet notes from memory

## Reading Rules

- skim the structure first
- read for definitions and logic, not for completion
- stop and test understanding after each section
- if a section is confusing, ask the AI to reteach it directly

## Coding Rules

- code before you feel perfectly ready
- keep the first version small and inspectable
- print shapes, intermediate values, and outputs
- compare expected behavior to actual behavior
- write what broke and how you fixed it

## Retention Rules

- end each day with 5 questions
- revisit weak topics within 48 hours
- if you cannot explain it simply, do not count it as learned
- if you cannot implement the core idea, slow down the roadmap
