# Source Selection Rules

Use these rules whenever the AI recommends study material for a topic.

## Goal

Pick the smallest set of high-quality sources that helps me understand, implement, and retain the topic.

## Preferred Source Order

1. one strong intuition source
2. one rigorous primary learning source
3. one implementation-oriented source
4. one optional reinforcement source

Do not overload the day with many resources.

## Source Quality Rules

- prefer canonical or widely trusted educators
- prefer sources with clear explanations over flashy summaries
- prefer one complete source over many fragmented ones
- prefer sources that match my current level
- prefer free and directly accessible sources when possible
- if a resource is too advanced, warn me before assigning it
- if a source is weak, outdated, or hype-driven, do not pick it

## Video Rules

- use videos for intuition, walkthroughs, and mental models
- do not assign long playlists unless I need them
- pick the exact lecture or section, not just the whole course
- explain why the video is worth my time

## Reading Rules

- use readings for definitions, math, and conceptual precision
- specify exact chapters, pages, or sections when possible
- if the best source is paid or unavailable, teach the concept directly instead

## Coding Resource Rules

- pair each topic with one implementation reference when useful
- prefer sources that show how the concept appears in code, tensors, data, training loops, or debugging
- if the code source hides too much detail, supplement it with direct explanation

## Output Format For Recommended Sources

For each source, include:

- name
- type: video, reading, docs, or code reference
- exact section to use
- why this source
- estimated time
- whether it is required or optional

## Daily Cap

For a normal study day, aim for:

- 1 to 2 required sources
- 1 optional reinforcement source
- 1 coding task

## Learning Fit Rules

If the topic is:

- math-heavy: prioritize derivation, worked examples, and handwritten notes
- implementation-heavy: prioritize coding, debugging, and experimentation
- systems-heavy: prioritize architecture diagrams, tradeoffs, and practical case studies
- weakly understood already: prioritize active recall and re-explanation over more new content
